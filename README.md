# 成行 SureGo MVP1.0

成行 SureGo 是面向大学生的同校搭子小程序，帮助用户围绕具体活动、任务和长期目标，快速发现搭子、建立连接、发起活动，并把饭局、运动、自习、游戏开黑、项目组队和长期搭子这些事真的约成。

它不是“刷人”的泛社交产品，而是一个 **活动/任务导向的搭子社交产品**：用户先看到明确的活动、任务或长期目标，再决定是否报名、提交意向、关注、私聊、进群或一起发起成行活动。

当前 `codex/big-refactor` 分支是 SureGo MVP1.0 的重构演示版本，重点跑通“成行活动 + 搭子连接 + 转化发局”的完整产品闭环。

## 产品主路径

### 1. 参加一个公开成行活动

```text
成行页 -> 活动详情 -> 报名/提交申请 -> 审核/候补/确认占位 -> 报名成功 -> 赴约信息 -> 到场核销
```

这条路径面向时间、地点、人数和规则相对明确的活动，例如饭局、运动、自习、桌游、校园主题活动和毕业季约拍。

### 2. 找搭子并建立连接

```text
搭子页 -> 搭子帖详情 -> 提交意向/申请认识/申请加入 -> 发帖人确认 -> 关注/私聊/群聊 -> 决定下一步
```

这条路径面向时间地点还未完全确定的需求，例如长期搭子、游戏搭子、自习搭子、项目队友、黑客松队友和任务型搭子。

### 3. 从搭子帖转成成行活动

```text
搭子帖 -> 多人提交意向 -> 发帖人确认时间地点 -> 转化发局 -> 公开活动 / 私密活动
```

这是 SureGo 的关键闭环：搭子不是停留在互相看看，而是要顺畅地建联，再顺畅地转成一次真实活动。

## MVP1.0 做什么

- 成行页：展示同校已经明确时间、地点、人数和规则的活动局。
- 搭子页：展示同校正在找的饭搭子、运动搭子、学习搭子、游戏搭子、项目组队和长期搭子。
- 发布入口：底部中间加号承接发布行为，区分“发布活动局”和“发布搭子帖”。
- 活动发布：支持填写时间、地点、人数、规则、报名问题和活动介绍。
- 搭子帖发布：支持约个时间、长期搭子、项目组队三类结构化需求。
- 报名与申请：支持报名、提交意向、审核、候补和确认占位的前端表达。
- 工作台：活动发起人管理报名、候补、成员、通知和核销；搭子帖发布者管理意向、私聊、群聊和转化发局。
- 通知与沟通：覆盖审核结果、候补补位、行前提醒、发帖意向、私聊消息和群聊消息。
- 学生认证：表达同校身份验证的价值和基础认证方式。
- 主题专区：包含毕业季约拍专题和 AI 黑客松组队专区。

## MVP1.0 暂不做什么

- 不做独立 App，先做小程序/小程序形态演示。
- 不做多学校，第一版只聚焦同校范围。
- 不接真实支付，只做确认占位、诚意金、门票等前端表达。
- 不做完整支付、退款、结算系统。
- 不做复杂推荐算法和完整信用体系。
- 不做完整长期搭子自动匹配算法。
- 不做大规模运营后台和多城市推广。

## 页面结构

```text
成行页
  -> 活动详情页
    -> 报名/申请页
      -> 确认占位页
        -> 报名成功页
          -> 我的页里的赴约/活动管理

搭子页
  -> 搭子帖详情页
    -> 提交意向/申请认识/申请加入
      -> 发帖人确认
        -> 关注/私聊/群聊
          -> 转化发局
            -> 公开活动 / 私密活动

+ 发布入口
  -> 发布活动局
    -> 发局工作台
  -> 发布搭子帖
    -> 发帖工作台

我的页
  -> 我的报名
  -> 我的赴约
  -> 我的发局
  -> 我的搭子帖
  -> 我的私聊/群聊
  -> 我的关注
  -> 基础资料和认证
```

## 仓库模块

```text
pages/home                成行首页、精选活动、我的局、推荐活动
pages/discover            城市/同校活动发现、分类筛选、搜索、日历入口
pages/graduation          毕业季活动专题
pages/partners            搭子广场、搜索、场景筛选
pages/partner             搭子帖详情、发布、工作台、会话
pages/hackathon           AI 黑客松组队专区、队伍详情、意向提交
pages/publish             发布入口，进入发活动或发搭子帖流程
pages/verify              学生认证基础展示
pages/activity            活动详情、创建、编辑、成员、报名
pages/manage              发局工作台、申请审核、签到核销
pages/participant         参与者活动面板和入场凭证
pages/payment             诚意金/门票确认占位流程
pages/messages            通知与消息列表
pages/ops                 试运营控制台、举报、用户与权限
pages/user                我的页、用户资料、他人主页
common/api                活动、搭子、报名、订单、消息、签到、用户等业务 API
common/mock               MVP 演示与本地开发用 Mock 数据
components/surego         SureGo 业务组件
```

## 技术栈

- uni-app
- Vue 3
- uniCloud
- uni-id 登录态
- uni-ui 与 SureGo 自定义业务组件
- 领域化 `common/api` 业务分层
- 试运营模式与本地开发 Mock fallback

## 运行方式

1. 使用 HBuilderX 导入项目。
2. 在 `manifest.json` 中确认微信小程序 AppID、H5 路由和 App 相关配置。
3. 绑定或创建 uniCloud 服务空间，并按项目需要配置 uni-id、数据库集合和云函数。
4. 在 HBuilderX 中选择运行到微信开发者工具、浏览器或 App 基座。
5. 开发阶段可结合本分支 Mock fallback 预览核心流程。

## 判断标准

MVP1.0 的判断标准是：

> 一个同校大学生能不能围绕一个具体活动、任务或长期目标，快速发现搭子需求，建立连接，并顺畅地转化成一次真实成行。

如果这条逻辑跑通，再继续做真实支付、完整后端、复杂信用、推荐算法、多校扩展和更完整的运营后台。
